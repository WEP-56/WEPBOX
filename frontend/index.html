常规设置
控制应用启动行为、窗口行为和异常提示。优先保留普通用户能直接理解的选项。
开机自动运行
写入 Windows `Run` 注册表项，随系统登录启动客户端。
应用启动后自动拉起内核
应用打开后自动启动 sing-box，无需再点首页总开关。
启动时隐藏主窗口
适合自启动场景。应用会直接进入后台，再通过托盘调出。
关闭窗口时隐藏到托盘
避免误关应用。真正退出请用托盘菜单里的“退出”。
异常退出时通知
内核异常退出并回退到安全状态时，在界面上弹出提示。
浅色模式
切换为浅色界面，适合白天或强光环境。
主题色
选择导航、按钮、开关和高亮状态使用的强调色。
代理设置
这里是会直接进入 `config.json` 或影响系统代理行为的核心设置。
本地 Mixed 端口
HTTP / SOCKS 混合入口。非 TUN 模式下也会被用于设置系统代理。
Clash API 端口
节点页、测速和运行探活依赖的控制端口。
Clash API Secret
用于本地 Clash API 鉴权。留空时后端会自动生成新的随机值。
允许局域网连接
把 Mixed 监听地址从 `127.0.0.1` 改为 `0.0.0.0`，便于局域网其他设备接入。
路由模式
规则 / 全局 / 直连 决定 `route.final` 和整体流量出口策略。
TUN 开关
接管系统流量。启用时需要管理员权限，并会关闭系统代理残留。
DNS 防泄漏
TUN 模式下劫持 DNS 请求，减少绕过 sing-box 的直连解析。
IPv6
切换 `prefer_ipv6` / `ipv4_only` 策略，并影响 TUN 地址生成。
规则管理
规则按“前置自定义 / 模式规则 / 内置分流 / 国内直连 / 非 CN 代理 / fallback”顺序生效。
规则模式 fallback
规则未命中时默认走 `proxy`，降低 TUN 规则模式下站点资源漏直连的概率。
增强应用规则集
启用 Google / Telegram / YouTube / Netflix / OpenAI 等常用服务规则集，并生成独立 selector 分流组。
广告规则拦截
接入 `geosite-category-ads-all`，在 DNS 和路由层拒绝广告域名。
当前内置规则集
这些规则集会随配置写入 `route.rule_set`，运行时由 sing-box 下载和更新。

TUN / FakeIP 规则保护
参考成熟客户端的实战策略，减少规则模式下 DNS 冷启动、FakeIP 误伤和 UDP 卡顿。
DNS 本地域名与 Windows 连通性检测域名直连解析 FakeIP 仅 TUN + Fake DNS 开启时写入，规则模式优先覆盖非 CN 域名 Route TUN 下阻断 QUIC，让浏览器回落 TCP Node 订阅节点域名使用直连 resolver，避免代理 DNS 冷启动依赖 Download 规则集下载 detour 跟随当前 fallback 策略
自定义前置规则
JSON 数组，插入到内置规则前。可用于临时修正域名、IP 或进程分流。
DNS / TUN 高级
暴露当前架构已经能真正落到配置链里的高级项；未接入项会明确标记。
自定义 DNS Servers
每行一个地址。第一个优先作为直连 DNS，第二个优先作为远端 DNS。
fake-dns
默认建议开启。关闭后 TUN 模式将不再生成 fakeip DNS server 和回流规则。
fake-ip IPv4 网段
默认 `198.18.0.0/15`，用于 fake-dns 返回的 IPv4 地址池。
fake-ip IPv6 网段
默认 `fc00::/18`，用于 fake-dns 返回的 IPv6 地址池。
TUN Interface Name
写入 `tun-in.interface_name`。Windows 网络设备列表里会显示这个名称。
TUN MTU
写入 `tun-in.mtu`。过小或过大都可能影响吞吐和兼容性。
auto_route
让 sing-box 自动下发路由。关闭前请确认你知道自己在做什么。
strict_route
更严格地限制流量必须按 TUN 路由走，可降低 DNS 泄漏风险，但可能影响国内直连稳定性。
私有网段排除列表
写入 `route_exclude_address`。每行一个 CIDR，默认保留局域网和链路本地地址。
UDP 加速
会把 UDP 连接优化写入 sing-box 路由与入站参数。
实验性 QUIC 优化
会对支持 QUIC 的 DNS 与出站参数写入更激进的传输字段。
订阅与转换
导入、转换和后台策略会集中到这里。当前只暴露已经能解释清楚的入口。
Converter URL
当本地无法识别订阅格式时，远程转换为 sing-box JSON 的备用服务。
订阅自动更新间隔
启动后后台定时检查远程订阅；到达间隔后自动刷新，失败会写入诊断事件日志。
测速计划
内核运行时按计划在后台测速，结果会缓存并用于节点排序与自动优选。
自动选择最快节点
测速计划完成后，会按分组切换到当前最低延迟的可用节点。
故障时自动切换
测速发现当前主代理组失败时，会自动切到当前可用的最佳节点。
当前订阅策略固定为：导入、刷新、删除订阅时，如果内核运行中，会自动重写配置并重载内核；手动导入的节点不支持远程刷新。
运行维护
提供排障、清理和诊断导出入口，优先补对稳定性与回收有帮助的能力。
